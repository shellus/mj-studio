# 对话模块调用链

## 全局概览

### 阶段一：请求发起与 SSE 建立

```mermaid
sequenceDiagram
    participant U as 前端 chat.vue
    participant C as useConversations
    participant API as POST /messages
    participant GE as globalEvents
    participant PL as global-events 插件
    participant SSE as GET /messages/[id]/stream

    U->>C: handleSendMessage()
    C->>API: sendMessage() → POST 请求
    API->>API: 创建用户消息 + AI 消息
    API->>GE: emitToUser('chat.message.created') ×2
    API-->>C: 返回 { userMessageId, assistantMessageId }
    GE-->>PL: SSE 推送 chat.message.created
    PL->>C: subscribeToMessageStream(messageId)
    C->>SSE: fetch(/api/messages/[id]/stream)

    Note over API: setImmediate → 启动流式生成（见阶段二）
```

### 阶段二：流式生成与推送

```mermaid
sequenceDiagram
    participant ST as streamingTask
    participant SC as streamingCache
    participant CP as ChatProvider
    participant UP as 上游 API
    participant SSE as GET /messages/[id]/stream
    participant GE as globalEvents

    ST->>SC: startStreamingSession()
    ST->>ST: 获取对话/助手/模型配置
    ST->>CP: getChatProvider(apiFormat)
    CP->>CP: createService(aimodel)
    ST->>UP: chatService.chatStream()

    loop 流式响应
        UP-->>ST: chunk (content/thinking/toolUse)
        ST->>SC: appendStreamingContent()
        SC-->>SSE: stream.push(content)
    end

    ST->>ST: 保存最终内容到数据库
    ST->>GE: emitToUser('chat.message.done')
```

## 分层详解

### 1. 前端发送消息

```mermaid
flowchart LR
    A["chat.vue<br/>handleSendMessage()"] --> B["useConversations<br/>sendMessage()"]
    B --> C["POST /api/conversations/[id]/messages"]
    C --> D["返回 { userMessageId, assistantMessageId }"]
```

| 文件 | 函数 | 职责 |
|------|------|------|
| `app/pages/chat.vue` | `handleSendMessage()` | UI 入口，收集输入内容和附件 |
| `app/composables/useConversations.ts:516` | `sendMessage()` | 发起 POST 请求，不等待 AI 响应 |

### 2. API 路由处理

```mermaid
flowchart TD
    A["POST /conversations/[id]/messages"] --> B[验证权限]
    B --> C[创建用户消息]
    C --> D["emitToUser('chat.message.created')"]
    C --> E[创建 AI 消息 status=created]
    E --> F["emitToUser('chat.message.created')"]
    E --> G["setImmediate → startStreamingTask()"]
    E --> H["立即返回 { userMessageId, assistantMessageId }"]
```

| 文件 | 关键逻辑 |
|------|---------|
| `server/api/conversations/[id]/messages.post.ts` | 创建两条消息后立即返回，通过 `setImmediate` 异步启动流式任务 |

> [!note] 异步解耦
> API 路由不等待 AI 生成完成，通过 `setImmediate` 将生成任务放入事件循环下一轮执行，前端通过 SSE 获取后续状态。

### 3. 流式生成核心

```mermaid
flowchart TD
    A["startStreamingTask()"] --> B["startStreamingSession()<br/>初始化缓存"]
    B --> C["获取对话/助手/模型/上游"]
    C --> D{是否压缩请求?}
    D -->|是| E["提取压缩区间消息"]
    D -->|否| F["从最后 compress-response 开始<br/>过滤特殊标记消息"]
    E --> G["updateMessageStatus → pending"]
    F --> G
    G --> H["getChatProvider(apiFormat)"]
    H --> I["chatProvider.createService(aimodel)"]
    I --> J["chatService.chatStream()"]
    J --> K["processChunk 循环"]
```

| 文件 | 函数 | 职责 |
|------|------|------|
| `server/services/streamingTask.ts:48` | `startStreamingTask()` | 流式生成主流程 |
| `server/services/streamingTask.ts:236` | `processChunk()` | 处理单个 chunk（content/thinking/toolUse/webSearch） |
| `server/services/streamingCache.ts:40` | `startStreamingSession()` | 初始化内存缓存 |
| `server/services/providerConnection.ts:6` | `resolveUpstreamConnection()` | 解析 apiKey + 代理 + baseUrl |

### 5. 流式内容处理

```mermaid
flowchart TD
    A["processChunk()"] --> B{chunk 类型}
    B -->|thinking| C["追加思考内容<br/>管理 thinking 标签"]
    B -->|webSearch| D["输出搜索状态<br/>代码块"]
    B -->|toolUse| E["加入 pendingToolCalls<br/>（不广播，见第 6 节）"]
    B -->|content| F["追加到 fullContent"]
    F --> G{首个内容块?}
    G -->|是| H["status → streaming<br/>更新预计首字时长"]
    G -->|否| I
    H --> I
    C & D & I --> J["appendStreamingContent()<br/>广播给所有 SSE 订阅者"]

    E -.->|"stopReason=tool_use"| K["进入 MCP 工具调用循环"]
```

### 6. MCP 工具调用循环

```mermaid
flowchart TD
    A["stopReason = tool_use"] --> B["保存当前 assistant 消息"]
    B --> C["创建 ToolCallRecord[]"]
    C --> D["广播工具调用状态"]
    D --> E{需要用户确认?}
    E -->|autoApprove| F["直接执行"]
    E -->|需确认| G["waitForToolConfirmation()"]
    G -->|批准| F
    G -->|拒绝| H["标记 cancelled"]
    F --> I["mcpClientManager.callTool()"]
    I --> J["更新 ToolCallRecord 状态"]
    J --> K["重新获取历史消息"]
    K --> L["chatService.chatStream()<br/>继续对话（不传 userMessage）"]
    L --> M{toolRounds < maxRounds?}
    M -->|是| A
    M -->|否| N["输出轮次限制提示"]
```

> [!warning] 最大轮次限制
> 工具调用有 `MCP_CLIENT_CONFIG.maxToolRounds` 轮次上限，超出后强制结束并输出提示。

### 7. 双通道同步机制

前端通过两个独立的 SSE 通道接收数据：

```mermaid
flowchart LR
    subgraph 后端
        ST["streamingTask"]
        SC["streamingCache"]
        GE["globalEvents"]
    end

    subgraph "通道 1：流式内容"
        SC -->|"stream.push(content)"| SSE1["GET /api/messages/[id]/stream"]
    end

    subgraph "通道 2：全局事件"
        GE -->|"SSE 推送"| SSE2["GET /api/events"]
    end

    subgraph 前端
        SSE1 -->|"增量内容"| MSG["消息内容实时更新"]
        SSE2 -->|"状态事件"| STATE["消息状态/对话列表更新"]
    end

    ST --> SC
    ST --> GE
```

| 通道 | 端点 | 传输内容 | 前端处理 |
|------|------|---------|---------|
| 流式内容 | `GET /api/messages/[id]/stream` | 增量文本 chunk | `subscribeToMessageStream()` 逐块追加 |
| 全局事件 | `GET /api/events` | 结构化事件 | `global-events.client.ts` 插件分发 |

### 8. 前端事件处理

```mermaid
flowchart TD
    A["global-events.client.ts 插件"] --> B{事件类型}
    B -->|chat.message.created| C["添加/更新消息到列表"]
    C --> D{assistant 且 status ∈ created/pending/streaming?}
    D -->|是| E["subscribeToMessageStream()"]
    B -->|chat.message.done| F["更新消息状态"]
    F --> G["取消流式订阅"]
    F --> H["更新模型预计时长"]
    B -->|chat.message.updated| I["更新消息内容/状态"]
    B -->|chat.message.deleted| J["从列表移除消息"]
```

| 文件 | 职责 |
|------|------|
| `app/plugins/global-events.client.ts:73` | 监听 `chat.message.created`，触发流式订阅 |
| `app/plugins/global-events.client.ts:126` | 监听 `chat.message.done`，结束流式状态 |
| `app/composables/useConversations.ts:198` | `subscribeToMessageStream()` — 建立 SSE 连接读取增量内容 |

### 9. 停止生成

```mermaid
sequenceDiagram
    participant U as 用户点击停止
    participant API as DELETE /messages/[id]/stop
    participant ST as stopStreamingTask
    participant SC as streamingCache
    participant SL as 流式循环

    U->>API: 请求停止
    API->>ST: stopStreamingTask(messageId)
    ST->>SC: abortController.abort()
    ST->>ST: cancelPendingToolCalls()
    ST->>ST: sleep 100ms

    alt Case A
        Note right of SL: 流式循环先完成
        SL->>SC: tryFinalizeSession() ok
        SL->>SL: 保存内容 + 广播 done
        ST->>SC: tryFinalizeSession() null
    else Case B
        Note right of ST: 流式循环未响应
        ST->>SC: tryFinalizeSession() ok
        ST->>ST: 保存已累积内容 + 广播 done
    end
```

> [!note] 竞态条件处理
> `tryFinalizeSession()` 通过 `finalized` 标志实现原子操作，确保停止时只有一方（流式循环或 stopStreamingTask）负责保存内容。
